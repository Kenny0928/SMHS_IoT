IoT 機器人實驗課  ·  ESP32 麥克納姆輪車平台

系統總覽

smhs_IoT_Robot 是一個以 ESP32 為核心的 IoT 機器人平台， 結合麥克納姆輪四驅底盤、多種環境感測器與 WiFi 無線控制， 讓學生透過循序漸進的練習，掌握嵌入式系統與物聯網的核心概念。

函式庫說明

所有硬體物件都定義在函式庫的 src/smhs_IoT_Robot.h 中。使用 Arduino IDE 匯入 ZIP 程式庫後，在程式最頂端加入：

#include <smhs_IoT_Robot.h>
Motor — 單顆馬達控制
Motor m1(PIN_A, PIN_B);        // 建立馬達物件，傳入兩個 PWM 腳位
m1.act(Motor::CW,  600);       // 順時針轉，速度 600 (0~1023)
m1.act(Motor::CCW, 600);       // 逆時針轉
m1.stop();                     // 煞車停止
GoSUMO — 四輪麥克納姆輪車控制
// 先建立四顆馬達，再傳入 GoSUMO
Motor m1(SMHS_HW::M1A, SMHS_HW::M1B);
Motor m2(SMHS_HW::M2A, SMHS_HW::M2B);
Motor m3(SMHS_HW::M3A, SMHS_HW::M3B);
Motor m4(SMHS_HW::M4A, SMHS_HW::M4B);
GoSUMO robot(&m1, &m2, &m3, &m4);

// 基礎運動
robot.act(GoSUMO::FORWARD,      570);  // 前進
robot.act(GoSUMO::BACKWARD,     570);  // 後退
robot.act(GoSUMO::TURN_LEFT,    570);  // 原地左轉
robot.act(GoSUMO::TURN_RIGHT,   570);  // 原地右轉

// 橫移 (麥克納姆輪特有)
robot.act(GoSUMO::STRAFE_LEFT,  570);  // 左橫移
robot.act(GoSUMO::STRAFE_RIGHT, 570);  // 右橫移

// 斜向運動 (麥克納姆輪特有)
robot.act(GoSUMO::DIAG_FL,      570);  // 前左斜向
robot.act(GoSUMO::DIAG_FR,      570);  // 前右斜向
robot.act(GoSUMO::DIAG_BL,      570);  // 後左斜向
robot.act(GoSUMO::DIAG_BR,      570);  // 後右斜向

robot.stop();                          // 全部煞車
stateLED — WS2812B 全彩狀態指示燈
stateLED robotLED(SMHS_HW::WSLED, 1);  // 腳位 2，共 1 顆 LED
robotLED.begin();                       // 初始化 (在 setup 中呼叫)
robotLED.setBrightness(60);            // 設定亮度 (0~255，建議 40~100)

// 全部 LED 同色 (最常用)
robotLED.fillColor(stateLED::RED);      // 紅色 (錯誤、攻擊、低電量)
robotLED.fillColor(stateLED::GREEN);    // 綠色 (安全、待機、就緒)
robotLED.fillColor(stateLED::BLUE);     // 藍色 (運行中)
robotLED.fillColor(stateLED::YELLOW);   // 黃色 (警告)
robotLED.fillColor(stateLED::PURPLE);   // 紫色 (特殊模式)
robotLED.fillColor(stateLED::CYAN);     // 青色 (自訂狀態)
robotLED.fillColor(stateLED::WHITE);    // 白色 (照明、啟動)
robotLED.fillColor(stateLED::OFF);      // 關閉

// 自訂 RGB 顏色
robotLED.fill(255, 128, 0);            // 直接指定 R, G, B 值 (0~255)

// 關閉所有 LED
robotLED.clear();

// 單顆控制 (多 LED 時使用)
robotLED.setColor(0, stateLED::RED);            // 第 0 顆 → 紅色
robotLED.setPixel(0, 255, 128, 0);             // 第 0 顆 → 自訂 RGB
Buzzer — 被動式蜂鳴器控制
Buzzer buzzer(SMHS_HW::BUZZER);   // 建立蜂鳴器物件，傳入腳位

// 基礎發聲
buzzer.tone(1000);                 // 發出 1000 Hz 音調 (持續到 noTone)
buzzer.tone(1000, 200);            // 發出 1000 Hz，音量 200 (0~255)
buzzer.noTone();                   // 停止發聲

// 警報音
buzzer.alarm();                    // 發出警報音效

// 內建系統音效 (非阻斷式，需在 loop() 搭配 update() 呼叫)
buzzer.soundBoot();                // 開機音效
buzzer.soundConnect();             // 連線成功
buzzer.soundError();               // 錯誤警告
buzzer.soundNotify();              // 提醒音效
buzzer.soundAttack();              // 攻擊/警報

// 播放自訂旋律 (需在 loop() 中持續呼叫 update())
Note melody[] = {
  {Pitch::Do_4, 300}, {Pitch::Mi_4, 300}, {Pitch::Sol_4, 600}
};
buzzer.playMelody(melody, 3);

// ⚠️ 使用背景播放時，必須在 loop() 中呼叫：
buzzer.update();

SMHS_HW::M1A   // GPIO 14 (馬達 M1 A 相)
SMHS_HW::WSLED // GPIO 2  (狀態 LED)

練習 00 · WS2812B 狀態指示燈

難度：★☆☆ 檔案：examples/ex00_WS2812B/ex00_WS2812B.ino

學習目標

  • 認識 WS2812B 全彩 LED 的特性：只需一條數位訊號線即可控制顏色
  • 學習使用 stateLED 類別的 fillColor()、clear() 與 setBrightness()
  • 練習以 delay() 控制燈效時間序列（入門版計時方式）

重要觀念

ℹ️ WS2812B 與一般 LED 的差異
普通 LED 只能控制亮滅；WS2812B 是「智慧型」LED，內建驅動晶片， 透過單一腳位的串列訊號即可獨立控制每顆 LED 的紅、綠、藍三色強度（各 0~255）， 理論上可混合出 1600 萬種顏色。

關鍵程式碼說明

#include <smhs_IoT_Robot.h>

stateLED led(SMHS_HW::WSLED, 1);  // 腳位 2，共 1 顆 LED

void setup() {
  led.begin();
  led.setBrightness(60);  // 亮度設 60，避免太亮刺眼 (0~255)
}

void loop() {
  led.fillColor(stateLED::RED);   // 亮紅色
  delay(500);
  led.clear();                    // 熄滅
  delay(500);

  led.fillColor(stateLED::GREEN); // 亮綠色
  delay(500);
  led.clear();
  delay(500);

  led.fillColor(stateLED::BLUE);  // 亮藍色
  delay(500);
  led.clear();
  delay(500);
}
💡 延伸挑戰
試著加入 YELLOW、PURPLE、CYAN、WHITE 等顏色， 或用 led.fill(r, g, b) 自訂 RGB 數值，做出漸變燈效。

練習 02 · 光敏電阻資料上傳 ThingSpeak

難度：★★☆ 檔案：examples/ex02_LightThingSpeak/ex02_LightThingSpeak.ino

學習目標

  • 學習 ESP32 連接 WiFi 的基本流程
  • 認識 ThingSpeak IoT 雲端平台，了解如何用 HTTP GET 上傳感測器資料
  • 學習使用 HTTPClient 函式庫發送 HTTP 請求
  • 學習用狀態 LED 顯示連線狀態（紅 = 連線中、綠 = 連線成功、藍 = 上傳中）
  • 理解用 millis() 控制上傳間隔（ThingSpeak 免費版最快每 15 秒一次）

事前準備

ℹ️ 取得 ThingSpeak API Key
1. 前往 thingspeak.com 註冊帳號
2. 建立新 Channel，記下 Write API Key
3. 將程式中的 writeApiKey 替換為你的 Key，ssid / password 填入 WiFi 資訊

LED 狀態說明

LED 顏色代表狀態
🔴 紅色WiFi 連線中 / 斷線重連中
🟢 綠色WiFi 已連線，等待下次上傳
🔵 藍色資料上傳至 ThingSpeak 中

關鍵程式碼說明

// WiFi 連線
WiFi.begin(ssid, password);
while (WiFi.status() != WL_CONNECTED) { delay(500); }

// HTTP GET 上傳資料
void uploadLight(int value) {
  HTTPClient http;
  String url = "http://api.thingspeak.com/update?api_key=";
  url += writeApiKey;
  url += "&field1=" + String(value);

  http.begin(url);
  int code = http.GET();  // 回傳 > 0 表示上傳成功
  http.end();
}

// loop() 中以 millis() 控制上傳頻率
if (millis() - lastUploadTime >= UPLOAD_INTERVAL) {
  lastUploadTime = millis();
  uploadLight(analogRead(PIN_LIGHT));
}
⚠️ ThingSpeak 上傳限制
ThingSpeak 免費帳號要求每次上傳間隔至少 15 秒。 程式中 UPLOAD_INTERVAL = 20000（20 秒）是合理的設定值。 若上傳太頻繁，ThingSpeak 會回傳 0 表示拒絕。
💡 延伸挑戰
試著同時上傳溫濕度資料到 field2、field3： 在 URL 後加入 &field2=溫度&field3=濕度，並在 ThingSpeak 的 Channel 設定中啟用對應欄位。

重要觀念

⚠️ DHT11 最快讀取速率限制
DHT11 每次讀取後需要至少 2 秒的冷卻時間。 若讀取頻率太快，函式會回傳 NaN（無效數字）。 程式中以 READ_INTERVAL = 2000 毫秒確保不超頻讀取。

關鍵程式碼說明

// 讀取溫濕度
float h = dht.readHumidity();
float t = dht.readTemperature();

// isnan() 檢查是否為無效數值
if (isnan(h) || isnan(t)) {
  Serial.println("DHT11 讀取失敗！");
  return;  // 直接跳出，不更新數值
}

temperature = t;
humidity = h;
💡 延伸挑戰
試著加入判斷：當溫度超過 30°C 時，讓狀態 LED 變成紅色；低於 25°C 時變成藍色。

接線方式

雨水模組腳位ESP32 腳位說明

ℹ️ 什麼是 Active Low？
雨水模組在「偵測到水分」時輸出 LOW（0），「乾燥」時輸出 HIGH（1）。 這與直覺相反，程式中用 isRaining = (rawValue == LOW) 處理這個邏輯反轉。

關鍵程式碼說明

// 設定腳位為數位輸入
pinMode(PIN_RAIN, INPUT);

// 讀取數位值 (回傳 HIGH=1 或 LOW=0)
int rawValue = digitalRead(PIN_RAIN);

// Active Low：LOW = 有雨；HIGH = 無雨
bool isRaining = (rawValue == LOW);
💡 延伸挑戰
試著結合 LED 燈效：偵測到有雨時 LED 閃爍藍色，無雨時維持綠色。

練習 05 · 光照感測器 (類比輸入)

難度：★★☆ 檔案：examples/ex05_Light/ex05_Light.ino

學習目標

  • 認識類比感測器的輸出特性：數值範圍 0 ~ 4095（12-bit ADC）
  • 學習使用 analogRead() 讀取電壓變化
  • 學習使用 map() 函式進行數值線性換算
  • 學習使用 constrain() 防止數值超出合理範圍
  • 理解 GPIO 34 等腳位為「純輸入」，不可用於輸出

接線方式

光敏電阻模組腳位ESP32 腳位說明
VCC3.3V電源供電
AO (類比輸出)GPIO 34類比光照訊號
GNDGND共地

重要觀念：ADC 解析度

ℹ️ ESP32 的 ADC 解析度
ESP32 的類比數位轉換器 (ADC) 解析度為 12 位元， 即輸入電壓 0V ~ 3.3V 對應數值 0 ~ 4095。
光線越強 → 光敏電阻阻值越低 → 分壓越高 → ADC 數值越大（越亮）。

關鍵程式碼說明

// 讀取類比值 (0 ~ 4095)
int lightRaw = analogRead(PIN_LIGHT);

// map(數值, 輸入最小, 輸入最大, 輸出最小, 輸出最大)
// 將 0~4095 線性換算為 0~100 的百分比
int lightPct = map(lightRaw, 0, 4095, 0, 100);

// constrain 確保結果不超出 0~100
lightPct = constrain(lightPct, 0, 100);
💡 延伸挑戰
試著加入自動夜燈功能：當光照強度低於 20% 時，讓 LED 亮白燈；超過 20% 時自動關閉。

看門狗機制說明

ℹ️ 為什麼需要看門狗？
網頁搖桿透過 WiFi 傳送指令。若 WiFi 突然斷線，車子會繼續以最後收到的指令行駛， 造成失控！

解決方法：讓搖桿「按住」時每 250ms 自動重送一次心跳訊號。 ESP32 如果連續 500ms 沒收到訊號，看門狗判定連線中斷，強制煞車。

關鍵程式碼說明

// loop() 中的看門狗邏輯
if (isMoving && (currentTime - lastCmdTime > CMD_TIMEOUT)) {
  robot.stop();     // 強制煞車
  isMoving = false;
  Serial.println("看門狗觸發：超時自動煞車！");
}

// 收到指令時更新心跳時間
lastCmdTime = millis();  // 重置看門狗計時器
// 網頁 JavaScript 的心跳連發邏輯
function startCmd(d){
  fetch('/cmd?dir='+d);  // 立即發送
  // 每 250ms 重送一次，維持「按住」狀態
  moveInterval = setInterval(()=>{ fetch('/cmd?dir='+d); }, 250);
}
function stopCmd(){
  clearInterval(moveInterval);  // 停止連發
  fetch('/cmd?dir=S');           // 發送煞車指令
}

練習 07 · 馬達接線前進測試

難度：★☆☆ 檔案：examples/ex07_MotorForwardTest/ex07_MotorForwardTest.ino

學習目標

  • 確認 M1、M2、M3、M4 四顆馬達是否接在正確位置
  • 確認每顆馬達 A/B 線方向是否能讓車體往前移動
  • 學會用 GoSUMO::FORWARD 做最基本的車體動作測試
安全提醒
燒錄後請先架高車體或讓輪子離地。程式倒數 3 秒後會持續前進， 確認方向正確後再放到地面測試。

測試判斷方式

觀察到的狀況可能原因建議檢查

常見問題

Q：燒錄後 Serial 沒有任何輸出？

請確認 Serial 監控視窗的鮑率設定為 115200，與程式中 Serial.begin(115200) 一致。

Q：WiFi 一直無法連線？
  • 確認 ssid 和 password 拼寫正確（區分大小寫）。
  • ESP32 只支援 2.4 GHz 頻段，不支援 5 GHz。手機熱點請確認設為 2.4 GHz。
  • 若使用手機熱點，確認熱點已開啟且在訊號範圍內。
Q：DHT11 Serial 顯示讀取失敗？
  • 確認 DATA 腳接在 GPIO 4，且有正確接地與供電 (3.3V)。
  • 確認程式中 READ_INTERVAL 設定為 2000 毫秒以上。
  • 部分劣質 DHT11 需在 DATA 與 VCC 之間接一顆 10kΩ 上拉電阻。
Q：光照感測器數值始終是 0 或 4095？
  • 確認訊號腳接在 GPIO 34（類比輸入），而非 GPIO 27（數位輸入）。
  • 部分光敏電阻模組有 AO（類比） 和 DO（數位） 兩個輸出腳，請確認使用的是 AO。
Q：網頁搖桿按鈕沒有反應，馬達不動？
  • 確認手機/電腦與 ESP32 在同一個 WiFi 網路下。
  • 確認在瀏覽器輸入的 IP 位址正確（從 Serial 監控視窗複製）。
  • 確認電池電量充足，馬達需要較大電流，USB 單獨供電可能不夠。
Q：車子自己停下來，沒有在控制的時候也會動？

這是正常的「看門狗」保護機制。按住搖桿按鈕時，瀏覽器每 250ms 自動送出心跳訊號維持移動； 放開後發送煞車指令，若網路延遲超過 500ms 看門狗也會自動煞車。
