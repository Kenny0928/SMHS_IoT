IoT 機器人實驗課  ·  ESP32 麥克納姆輪車平台

系統總覽

smhs_IoT_Robot 是一個以 ESP32 為核心的 IoT 機器人平台， 結合麥克納姆輪四驅底盤、多種環境感測器與 WiFi 無線控制， 讓學生透過循序漸進的練習，掌握嵌入式系統與物聯網的核心概念。

函式庫說明

所有硬體物件都定義在函式庫的 src/smhs_IoT_Robot.h 中。使用 Arduino IDE 匯入 ZIP 程式庫後，在程式最頂端加入：

#include <smhs_IoT_Robot.h>
Motor — 單顆馬達控制
Motor m1(PIN_A, PIN_B);        // 建立馬達物件，傳入兩個 PWM 腳位
m1.act(Motor::CW,  600);       // 順時針轉，速度 600 (0~1023)
m1.act(Motor::CCW, 600);       // 逆時針轉
m1.stop();                     // 煞車停止
GoSUMO — 四輪麥克納姆輪車控制
// 先建立四顆馬達，再傳入 GoSUMO
Motor m1(SMHS_HW::M1A, SMHS_HW::M1B);
Motor m2(SMHS_HW::M2A, SMHS_HW::M2B);
Motor m3(SMHS_HW::M3A, SMHS_HW::M3B);
Motor m4(SMHS_HW::M4A, SMHS_HW::M4B);
GoSUMO robot(&m1, &m2, &m3, &m4);

robot.act(GoSUMO::FORWARD,      570);  // 前進
robot.act(GoSUMO::BACKWARD,     570);  // 後退
robot.act(GoSUMO::TURN_LEFT,    570);  // 原地左轉
robot.act(GoSUMO::TURN_RIGHT,   570);  // 原地右轉
robot.act(GoSUMO::STRAFE_LEFT,  570);  // 左橫移
robot.act(GoSUMO::STRAFE_RIGHT, 570);  // 右橫移
robot.stop();                          // 全部煞車
stateLED — WS2812B 全彩狀態指示燈
stateLED robotLED(SMHS_HW::WSLED, 1);  // 腳位 2，共 1 顆 LED
robotLED.begin();                       // 初始化 (在 setup 中呼叫)

robotLED.fillColor(stateLED::RED);      // 全亮紅色
robotLED.fillColor(stateLED::GREEN);    // 全亮綠色
robotLED.fillColor(stateLED::BLUE);     // 全亮藍色
robotLED.fillColor(stateLED::YELLOW);   // 全亮黃色
robotLED.fillColor(stateLED::OFF);      // 關閉

練習 01 · DHT11 溫濕度感測器

難度：★☆☆ 檔案：examples/ex01_DHT11/ex01_DHT11.ino

學習目標

  • 認識 DHT11 溫濕度感測器的接線方式與工作原理
  • 學習使用 DHT 函式庫的 readTemperature() 與 readHumidity()
  • 學習用 isnan() 檢查感測器是否正常回傳數值
  • 認識非阻斷式計時 (millis())，不使用 delay() 定時讀取

接線方式

DHT11 腳位ESP32 腳位說明
VCC3.3V電源供電
DATAGPIO 4資料訊號
GNDGND共地

重要觀念

⚠️ DHT11 最快讀取速率限制
DHT11 每次讀取後需要至少 2 秒的冷卻時間。 若讀取頻率太快，函式會回傳 NaN（無效數字）。 程式中以 READ_INTERVAL = 2000 毫秒確保不超頻讀取。

關鍵程式碼說明

// 讀取溫濕度
float h = dht.readHumidity();
float t = dht.readTemperature();

// isnan() 檢查是否為無效數值
if (isnan(h) || isnan(t)) {
  Serial.println("DHT11 讀取失敗！");
  return;  // 直接跳出，不更新數值
}

temperature = t;
humidity = h;
💡 延伸挑戰
試著加入判斷：當溫度超過 30°C 時，讓狀態 LED 變成紅色；低於 25°C 時變成藍色。

練習 02 · 雨水感測器 (數位輸入)

難度：★☆☆ 檔案：examples/ex02_Rain/ex02_Rain.ino

學習目標

  • 認識數位感測器的工作原理：只輸出 HIGH 或 LOW 兩種狀態
  • 學習使用 pinMode() 設定腳位為輸入，及 digitalRead() 讀取狀態
  • 理解「Active Low」設計邏輯：有水 → LOW；無水 → HIGH
  • 學習使用感測器模組上的旋鈕調整靈敏度（觸發臨界值）

接線方式

雨水模組腳位ESP32 腳位說明
VCC3.3V電源供電
DO (數位輸出)GPIO 27數位感測訊號
GNDGND共地
ℹ️ 什麼是 Active Low？
雨水模組在「偵測到水分」時輸出 LOW（0），「乾燥」時輸出 HIGH（1）。 這與直覺相反，程式中用 isRaining = (rawValue == LOW) 處理這個邏輯反轉。

關鍵程式碼說明

// 設定腳位為數位輸入
pinMode(PIN_RAIN, INPUT);

// 讀取數位值 (回傳 HIGH=1 或 LOW=0)
int rawValue = digitalRead(PIN_RAIN);

// Active Low：LOW = 有雨；HIGH = 無雨
bool isRaining = (rawValue == LOW);
💡 延伸挑戰
試著結合 LED 燈效：偵測到有雨時 LED 閃爍藍色，無雨時維持綠色。

練習 03 · 光照感測器 (類比輸入)

難度：★★☆ 檔案：examples/ex03_Light/ex03_Light.ino

學習目標

  • 認識類比感測器的輸出特性：數值範圍 0 ~ 4095（12-bit ADC）
  • 學習使用 analogRead() 讀取電壓變化
  • 學習使用 map() 函式進行數值線性換算
  • 學習使用 constrain() 防止數值超出合理範圍
  • 理解 GPIO 34 等腳位為「純輸入」，不可用於輸出

接線方式

光敏電阻模組腳位ESP32 腳位說明
VCC3.3V電源供電
AO (類比輸出)GPIO 34類比光照訊號
GNDGND共地

重要觀念：ADC 解析度

ℹ️ ESP32 的 ADC 解析度
ESP32 的類比數位轉換器 (ADC) 解析度為 12 位元， 即輸入電壓 0V ~ 3.3V 對應數值 0 ~ 4095。
光線越強 → 光敏電阻阻值越低 → 分壓越高 → ADC 數值越大（越亮）。

關鍵程式碼說明

// 讀取類比值 (0 ~ 4095)
int lightRaw = analogRead(PIN_LIGHT);

// map(數值, 輸入最小, 輸入最大, 輸出最小, 輸出最大)
// 將 0~4095 線性換算為 0~100 的百分比
int lightPct = map(lightRaw, 0, 4095, 0, 100);

// constrain 確保結果不超出 0~100
lightPct = constrain(lightPct, 0, 100);
💡 延伸挑戰
試著加入自動夜燈功能：當光照強度低於 20% 時，讓 LED 亮白燈；超過 20% 時自動關閉。

看門狗機制說明

ℹ️ 為什麼需要看門狗？
網頁搖桿透過 WiFi 傳送指令。若 WiFi 突然斷線，車子會繼續以最後收到的指令行駛， 造成失控！

解決方法：讓搖桿「按住」時每 250ms 自動重送一次心跳訊號。 ESP32 如果連續 500ms 沒收到訊號，看門狗判定連線中斷，強制煞車。

關鍵程式碼說明

// loop() 中的看門狗邏輯
if (isMoving && (currentTime - lastCmdTime > CMD_TIMEOUT)) {
  robot.stop();     // 強制煞車
  isMoving = false;
  Serial.println("看門狗觸發：超時自動煞車！");
}

// 收到指令時更新心跳時間
lastCmdTime = millis();  // 重置看門狗計時器
// 網頁 JavaScript 的心跳連發邏輯
function startCmd(d){
  fetch('/cmd?dir='+d);  // 立即發送
  // 每 250ms 重送一次，維持「按住」狀態
  moveInterval = setInterval(()=>{ fetch('/cmd?dir='+d); }, 250);
}
function stopCmd(){
  clearInterval(moveInterval);  // 停止連發
  fetch('/cmd?dir=S');           // 發送煞車指令
}
💡 延伸挑戰
試著加入橫移功能：在搖桿介面新增「左橫移」和「右橫移」按鈕， 使用 GoSUMO::STRAFE_LEFT 和 GoSUMO::STRAFE_RIGHT。

常見問題

Q：燒錄後 Serial 沒有任何輸出？

請確認 Serial 監控視窗的鮑率設定為 115200，與程式中 Serial.begin(115200) 一致。

Q：WiFi 一直無法連線？
  • 確認 ssid 和 password 拼寫正確（區分大小寫）。
  • ESP32 只支援 2.4 GHz 頻段，不支援 5 GHz。手機熱點請確認設為 2.4 GHz。
  • 若使用手機熱點，確認熱點已開啟且在訊號範圍內。
Q：DHT11 Serial 顯示讀取失敗？
  • 確認 DATA 腳接在 GPIO 4，且有正確接地與供電 (3.3V)。
  • 確認程式中 READ_INTERVAL 設定為 2000 毫秒以上。
  • 部分劣質 DHT11 需在 DATA 與 VCC 之間接一顆 10kΩ 上拉電阻。
Q：光照感測器數值始終是 0 或 4095？
  • 確認訊號腳接在 GPIO 34（類比輸入），而非 GPIO 27（數位輸入）。
  • 部分光敏電阻模組有 AO（類比） 和 DO（數位） 兩個輸出腳，請確認使用的是 AO。
Q：網頁搖桿按鈕沒有反應，馬達不動？
  • 確認手機/電腦與 ESP32 在同一個 WiFi 網路下。
  • 確認在瀏覽器輸入的 IP 位址正確（從 Serial 監控視窗複製）。
  • 確認電池電量充足，馬達需要較大電流，USB 單獨供電可能不夠。
Q：車子自己停下來，沒有在控制的時候也會動？

這是正常的「看門狗」保護機制。按住搖桿按鈕時，瀏覽器每 250ms 自動送出心跳訊號維持移動； 放開後發送煞車指令，若網路延遲超過 500ms 看門狗也會自動煞車。
